# 工具调用样式调整实施记录

## 背景

原工具调用样式偏“调试面板”：字号更小、状态标签和边框较重，参数与结果区域像独立卡片。
在对话流里它和“思考”块的轻量展开样式不一致，阅读时会显得突兀。

## 本阶段变更

- 工具调用触发器改为接近“思考”块的单行样式：
  - `text-sm`、`text-muted-foreground`、悬停变亮。
  - 使用同样的下拉箭头旋转方向。
  - 状态从醒目的 badge 改为轻量文本和小图标。
- 工具调用展开区改为和“思考”内容一致的 `mt-4 text-sm` 节奏。
- 参数和结果仍保留滚动限制，避免大输出撑高整段对话。
- 将工具状态文案改为中文，和当前前端主要界面语言保持一致。

## 原因

这不是给旧样式临时盖一层颜色，而是把工具调用的结构层级改成和思考块一致：
触发器负责说明“调用了什么工具、当前状态是什么”，展开内容负责承载参数和结果。

## 验证

- 已执行：`cd web && npm run build`
- 结果：通过。Vite 仍输出部分 chunk 超过 500 kB 的体积警告，属于现有打包体积提示。
