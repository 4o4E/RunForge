# 前端路由化实施记录

## 背景

原逻辑把当前会话 ID 只保存在 `App` 的 React 内存状态中。用户刷新页面、复制链接或使用
浏览器前进后退时，前端无法从地址恢复当前会话，只能回到“新会话”状态。

## 本阶段变更

- 新增轻量路由工具，将前端会话页映射为稳定地址：
  - `/chat/new` 表示新会话页。
  - `/chat/:threadId` 表示指定会话页，`threadId` 是 path value（路径变量）。
  - `?draft=...` 保存未发送的文本草稿，属于 query param（查询参数）。
- `App` 改为从 URL 读取当前会话，并监听浏览器前进/后退。
- 选择会话、新建会话、首次发送后端创建会话时，都会同步更新 URL。
- 刷新 `/chat/:threadId` 时重新从后端加载该会话历史；刷新带 `draft` 的地址时恢复输入框草稿。
- 附件不放入 URL，因为浏览器出于安全原因不允许恢复本地文件选择。

## 原因

路由只承载“定位信息”和轻量 UI 状态，消息内容仍使用后端持久化历史。这样地址可以分享、
刷新可以恢复，也不会把长对话内容塞进 URL。

## 验证

- 已执行：`cd web && npm run build`
- 结果：通过。Vite 仍输出部分 chunk 超过 500 kB 的体积警告，属于现有打包体积提示。
- 已执行：`curl -I http://127.0.0.1:3000/chat/new`
- 已执行：`curl -I 'http://127.0.0.1:3000/chat/test-thread?draft=...'`
- 结果：两个直达路由都返回 `200 OK` 和前端入口 HTML。
