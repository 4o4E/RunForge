# 前端代理后端接口实施记录

## 背景

前端开发服务需要代理后端接口，避免浏览器直接跨域访问后端。原逻辑已经代理了
`/api` 和 `/ws`，但后端地址固定为 `localhost:8080`，并且没有覆盖后端健康检查接口
`/health`。

## 本阶段变更

- 将 `web/vite.config.ts` 的代理目标改为可配置：
  - 优先读取 `VITE_BACKEND_ORIGIN`
  - 其次读取 `BACKEND_ORIGIN`
  - 默认使用 `http://localhost:8080`
- 保留 `/api` 代理，用于前端访问后端 REST 接口。
- 保留 `/ws` 代理并启用 WebSocket 转发，用于前端订阅 run 事件流。
- 新增 `/health` 代理，用于前端或调试页面检查后端健康状态时走同一代理入口。
- 开启 `changeOrigin`，让代理请求的 Host 与目标后端保持一致，减少反向代理或后端校验场景下的问题。

## 原因

这不是临时绕过跨域问题，而是把前端开发入口统一成同源访问：

- 浏览器只访问前端开发服务，例如 `http://localhost:3000/api/threads`。
- Vite 开发服务器在服务端侧把请求转发给后端，例如 `http://localhost:8080/api/threads`。
- 前端代码继续使用相对路径，不需要知道后端实际地址。

这样本地开发、远程后端联调、容器环境都可以通过环境变量切换后端地址。

## 验证

- 已执行：`npm run build -w web`
- 结果：通过。Vite 输出了部分 chunk 超过 500 kB 的体积警告，属于现有打包体积提示，不影响本次代理配置。
